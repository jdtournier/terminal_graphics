Detailed Description

The main class responsible for plotting data to the terminal.

To produce plots on the terminal, an instance of this class needs to be created, which sets up the size (in pixels) of the plot (or uses the default size if unspecified). Other public methods can then be used to add plots or text to the figure, or manipulate other properties such a limits and tick spacing. Finally, the show() method can be invoked to render the plot (note that this will implicitly be invoked in the Figure class destructor, and rarely needs to be invoked directly).

One-line plot

There are two main ways to produce a plot. The simplest is to use a one-line call to create an unnamed instance, and call all the required methods one after the other. All the relevant methods return a reference to the figure object, allowing them to be used for method chaining. For example:

termviz::figure()
.plot (x, y1)
.plot (x, y2)
.text ("my plot", 5, 5);

Since the destructor is called as soon as the line finishes, there is no need to add the final .show() call.

Note that we are using the termviz::figure() convenience call in the above example (though the termviz::Figure() constructor can be invoked directly if preferred).

long form

The second approach is to create a named instance of the termviz::Figure, call methods as required, and finally call the show() method (or allow the instance to go out of scope). This is useful in cases where the number of plots to add is not known at compile-time, for example. To illustrate:

std::vector<std::vector<float>> list_of_data = ...;
auto fig = termviz::figure (1024, 256).grid (false, true);
for (const auto& data : list_of_data)
fig.plot (data);
fig.xlim (0, 10);
fig.show();
